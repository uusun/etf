接口自检与部署说明

这个页面如何获取净值？

页面优先通过 https://fund.eastmoney.com/pingzhongdata/{基金代码}.js 读取历史单位净值； 盘中估算模式会再尝试通过 https://fundgz.1234567.com.cn/js/{基金代码}.js 读取估算净值。 因为是 GitHub Pages 静态网页，无法放后端代理，所以这里采用 JSONP / script 标签方式规避浏览器 CORS 限制。

如果某个接口被浏览器、网络或对方站点策略拦截，该基金会自动退回到本文件内置的基准净值与市值，并在“接口状态”列标记。

上传 GitHub Pages 的最简单方法
  1. 新建一个 GitHub 仓库，例如 fund-monitor。
  2. 把本文件改名或保持为 index.html，上传到仓库根目录。
  3. 进入仓库 Settings → Pages → Branch 选择 main / root。
  4. 打开 GitHub Pages 给出的地址即可。

注意：如果仓库或 Pages 是公开的，你的基金代码、份额、成本、市值会被公开看到。
